# Weekly Default Template

This template defines the full weekly summary format. Generated by aggregating daily summaries from the past week.

---

## Output Format Specification

### Header

```markdown
# Weekly Work Summary - Week {WW} ({Month Day} - {Month Day}, {Year})
```

### week_at_a_glance

Compact metrics box summarizing the week.

```markdown
## Week at a Glance

- Tasks: {N} completed across {project_count} projects
- Commits: {N} (+{added}/-{removed} lines)
- JIRA: {N} tickets tracked, {N} status changes
- Meetings: {total_hours}h ({meeting_count} meetings)
- Focus Time: ~{hours}h ({percentage}% average)
```

Only show lines for data sources that appear in dailies.

### major_accomplishments

Grouped by project. Synthesize related daily items into coherent narratives.

```markdown
## Major Accomplishments

### {Project Name}

- {Accomplishment synthesized from multiple daily items}
- {Another accomplishment}
- **Impact**: {One-line impact statement}

### {Another Project}

- {Accomplishment}
```

**Key rule**: Synthesize, don't repeat. Combine "Monday: created template", "Tuesday: documented template", "Wednesday: launched template" into "Template system created, documented, and launched."

### velocity_metrics

Detailed breakdown of code and task velocity.

```markdown
## Velocity Metrics

**Code Activity**:
- Commits/day: {avg} (Mon: {n}, Tue: {n}, Wed: {n}, Thu: {n}, Fri: {n})
- Lines changed: +{added} / -{removed} net
- Files modified: {N} across {repo_count} repos
- Peak day: {day} ({N} commits)

**Task Completion**:
- Tasks/day: {avg}
- Total completed: {N}
- High-priority: {N} ({percentage}%)

**Project Distribution**:
- {Project}: {percentage}%
- {Project}: {percentage}%
```

### time_allocation

Daily breakdown of meeting vs focus time.

```markdown
## Time Allocation

**Weekly Totals**:
- Focus Time: {hours}h ({percentage}%)
- Meetings: {hours}h ({percentage}%)

**Meeting Breakdown by Type**:
- 1:1s: {hours}h ({count} meetings)
- Planning/Refinement: {hours}h ({count})
- Reviews/Demos: {hours}h ({count})
- Standups: {hours}h ({count})

**Daily Patterns**:
- High-meeting days: {days with most meetings}
- High-focus days: {days with most focus time}
```

### jira_progress

JIRA tickets grouped by status movement this week.

```markdown
## JIRA Ticket Progress

**Completed This Week**:
- {TICKET-ID} - {Summary}

**Advanced (status change)**:
- {TICKET-ID} - {Summary} ({old_status} -> {new_status})

**Still In Progress**:
- {TICKET-ID} - {Summary}

**Blocked**:
- {TICKET-ID} - {Summary} ({blocker reason})
```

### blockers_status

Track blockers resolved and still open.

```markdown
## Blockers

**Resolved This Week**: {N}
- {Description} (resolved {day})

**Still Open**: {N}
- {Description} (estimated resolution: {date/note})
```

### project_health

Assessment of each active project.

```markdown
## Project Health

### Strong
- **{Project}**: {Brief assessment}

### Needs Attention
- **{Project}**: {Brief assessment}

### Blocked
- **{Project}**: {Brief assessment}
```

Criteria:
- **Strong**: 2+ tasks done, active commits, no blockers
- **Needs Attention**: Some progress but challenges
- **Blocked**: No progress or explicit blockers

### key_decisions

Decisions and outcomes from meetings and discussions this week.

```markdown
## Key Decisions & Outcomes

- **{Decision topic}** ({source: meeting/Slack/etc.}): {Outcome}
```

### next_week

Forward-looking priorities.

```markdown
## Next Week Priorities

**Must Complete**:
1. {Deadline-driven item}
2. {High-priority item}

**Focus Areas**:
- {Project}: {percentage}% ({description})

**Key Meetings**:
- {Day}: {Meeting} ({significance})
```

### velocity_comparison

Week-over-week trend analysis.

```markdown
## Velocity Comparison

**Week {N} vs Week {N-1}**:
- Tasks completed: {N} ({change}%)
- Commits: {N} ({change}%)
- Meeting hours: {N}h ({change}%)
- Focus time: {N}h ({change}%)

**Trend**: {Up/Down/Stable} - {One-line assessment}
```

### insights

Patterns and recommendations.

```markdown
## Insights & Patterns

**What Worked Well**:
- {Pattern that led to good outcomes}

**What Could Improve**:
- {Area for improvement}

**For Next Week**:
- {Specific recommendation}
```

### Footer

```markdown
---
Generated: {timestamp} {timezone}
Week: {WW} of {YYYY}
Daily summaries processed: {N} ({days covered})
Previous week comparison: Week {N-1}
```
